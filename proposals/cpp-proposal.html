Eclipse Tools C/C++Subproject Proposal

Description

The C for Eclipse Subproject is building a fully functional C and C++ Integrated Development Environment (IDE) for the Eclipse platform. Our focus is development on Linux for deployment on Linux, but we are interested in participation from others who would like to extend our work in other directions.e.g. a Windows client, targeting Unix(R) or embedded platforms, wizards for developing applications that use particular library or database or messaging APIs, or extension to other languages.

We are proposing this code base form the initial code for the open source C/C++ project hosted on eclipse.org under the eclipse tools project. The project goal would be the completion of a full featured C/C++ environment for all platforms. We are looking for contributions from the open source community in the areas of testers, developers and general users who can help us ensure that the C tools work well on all the Eclipse platforms.

Our first release function includes:

  • Support for the "autoconf" and "automake" utilities to define new projects or in existing projects imported into the environment. However, use of these tools is not mandatory, and your build may consist of hand-generated makefiles, Perl scripts, or whatever, as long as it can be launched from a command line Easily configurable error message parser, preconfigured for gcc and g++.
  • Code browsing, navigation, automatic code completion provided by an integrated C/C++ parser and program database.
  • Integrated debugger GUI using gdb, reusing many of the familiar Eclipse debug views but adding new ones for registers, storage, and a debugger console that provides access to the underlying gdb command line interface for advanced or esoteric debugger function.
  • Integrated access to the platform man, info, and HTML help. Select a keyword in the editor and hit F1 to search, results are displayed using the Gnome help browser or Konqueror according to your preference.
  • Support for projects that are remote, on another Linux system, as well as local development on your Linux desktop or laptop. Projects can be replicated between systems, so you can work locally for a while then replicate to a second Linux system where you rebuild and remotely edit and debug from within the IDE. The mechanism supporting this is extensible and only requires a Java 1.3 runtime on the remote Linux system, in addition to the underlying system utilities (gmake, gcc, gdb, perl) that it invokes.

Status

Our work is now at what we describe as a 0.5 level, meaning it isn't yet ready for production use but has progressed to the point where it will be of interest to the Eclipse developer community. Inconsistent or incomplete function, scalability, and stability problems may occur.

Platforms

The plugins are written in Java with no native code and thus may be ported to any platform supported by Eclipse. However, some Linux specific features prereq other tools that may not be found on non-Linux systems.

Content

We provide a total of 15 plugins and packages:

  1. com.ibm.cpp.ui - contributions of a C++ Perspective, views,and actions
  2. com.ibm.cpp.miners - remotable actors for build, launch, and project management
  3. com.ibm.cpp.miners.parser - the remotable C/C++ source parser
  4. com.ibm.cpp.docs.user - online Eclipse documentation
  5. com.ibm.debug - client-side extensions for debugging non-Java applications
  6. com.ibm.debug.doc.user - Eclipse documentation for the above
  7. com.ibm.debug.gdbPicl - server-side wrapper for GDB
  8. com.ibm.dstore.core - base Data Store metadata database implementation
  9. com.ibm.dstore.extra - client-side Data Store extensions
  10. com.ibm.dstore.extra.server - server-side Data Store extensions
  11. com.ibm.dstore.hosts - remote system support
  12. com.ibm.dstore.miners - remotable actors for generic command invocation
  13. com.ibm.dstore - ui Data Store related general views and actions
  14. com.ibm.linux.help - provides access to Linux platform-supplied help and reference information
  15. com.ibm.lpex - a Java implementation of the IBM Live Parsing Editor

Development plan

We are still in the process of defining our development plan however the major items for the first half of 2002 include:

  • rename our packages to use the org.eclipse prefix
  • make it easier to work with the components in the Eclipse Plugin Development Environment
  • add debugger function like an interlisted source/assembly view,
  • exceptions, conditional and deferred breakpoints
  • improve the performance and scalability of the parser and datastore
  • simplify the installation and invocation of the server-side components
  • begin integrating profiler support
  • improve stability

If you want to contribute to the development of this plugin, we welcome the opportunity to work with you. The plan will be updated to reflect the committments made by contributors to this projects.
